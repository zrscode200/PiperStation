# Review

Use a two-stage review:

1. Did we build the right thing?
2. Did we build it well?

Prioritize findings by severity. Anchor findings to concrete files, lines, behavior, missing verification, or drift from the request/spec/plan. The review is read-only unless the user asks for fixes or Ralph is applying verified review-gate findings.
